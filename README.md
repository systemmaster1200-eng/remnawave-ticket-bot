# Remnawave Ticket Bot

Telegram бот для менеджеров, позволяющий получать полную информацию о пользователях и их подписках из API Remnawave.

## Возможности

- 🔍 Поиск пользователей по Telegram ID или username
- 📊 Полная информация о пользователе (UUID, статус, трафик, даты)
- 🔗 Информация о подписке (статус, трафик)
- 📱 Привязанные устройства (HWID)
- 🔒 Доступ только для авторизованных менеджеров
- 💬 **Поддержка:** клиенты пишут в бота → каждому менеджеру приходит сообщение с данными из API и кнопкой «Ответить»; менеджер отвечает прямо в боте — ответ уходит клиенту
- 🐳 Работает в Docker контейнере

## Требования

- Python 3.11+
- Docker (для запуска в контейнере)
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather))
- Доступ к API Remnawave с токеном авторизации

## Установка и запуск

### 1. Клонирование и настройка

```bash
git clone https://github.com/systemmaster1200-eng/remnawave-ticket-bot.git
cd /opt/remnawave-ticket-bot
cp .env.example .env
```

### 2. Настройка переменных окружения

Отредактируйте файл `.env` и укажите:

- `BOT_TOKEN` - токен вашего Telegram бота
- `REMNAWAVE_API_URL` - URL API Remnawave (например, `https://api.remnawave.example.com`)
- `REMNAWAVE_API_TOKEN` - токен авторизации для API Remnawave
- `ALLOWED_MANAGER_IDS` - список ID менеджеров через запятую (например, `123456789,987654321`)
- `MINI_APP_DOMAIN` - домен мини-приложения (например, `mini.stealthnet.app` УТОЧНЕНИЕ - ставить на чистый сервер или рядом с панелью но тогда без мини апп) 
- `CERTBOT_EMAIL` - email для Let's Encrypt (нужен для автоматического SSL при запуске nginx)

### Подключение Groq (ИИ-поддержка)

Бот может автоматически отвечать клиентам в поддержке с помощью ИИ (Groq по умолчанию). Чтобы включить:

1. **Получите API-ключ Groq:** зарегистрируйтесь на [console.groq.com](https://console.groq.com), создайте проект и скопируйте API Key (начинается с `gsk_...`).

2. **Добавьте в `.env`:**
   ```bash
   GROQ_API_KEY=gsk_ваш_ключ_здесь
   GROQ_MODEL=llama-3.1-8b-instant
   ```
   Вместо `GROQ_API_KEY` можно использовать `AI_SUPPORT_API_KEY` — оба варианта поддерживаются.

3. **Опционально:**
   - `GROQ_MODELS` — список моделей через запятую для ротации (например, при лимитах). Указывайте только модели, доступные в вашем проекте в [Model Permissions](https://console.groq.com/settings/project/limits).
   - `GROQ_VISION_MODEL` — модель для обработки фото от клиентов (например, `llama-3.2-90b-vision-preview`). Если не задана, фото не отправляются в ИИ.
   - `GROQ_PROXY` — HTTP-прокси (например, `http://127.0.0.1:8080`), если с вашего сервера к api.groq.com идёт 403.
   - Несколько ключей (fallback): `AI_SUPPORT_API_KEYS=gsk_1,gsk_2`.

4. **Проверка при старте:** при запуске бот проверяет ключ Groq. Если ключ не задан или неверный, в логах будет предупреждение, автоответы ИИ работать не будут (менеджеры по-прежнему смогут отвечать вручную).

Альтернативный провайдер — **Gemini**: задайте `AI_PROVIDER=gemini` (или `AI_SUPPORT_API_TYPE=gemini`) и укажите `GEMINI_API_KEY` и `GEMINI_MODEL` в `.env`.

### 3. Запуск через docker-compose (рекомендуется: бот + мини-приложение + nginx + SSL)

На сервере должен быть открыт порт 80 и 443, домен `MINI_APP_DOMAIN` должен указывать на IP сервера.

```bash
# В .env укажите реальный email для CERTBOT_EMAIL
docker compose up -d --build
```

При первом запуске nginx автоматически получит SSL-сертификат Let's Encrypt для домена из `MINI_APP_DOMAIN`. Продление сертификата выполняется по крону раз в сутки.

Мини-приложение будет доступно по адресу `https://<MINI_APP_DOMAIN>`. В боте кнопка «Приложение» и кнопка в /start откроют этот URL.

### 4. Запуск в Docker (только бот)

```bash
# Сборка образа
docker build -t remnawave-manager-bot .

# Запуск контейнера
docker run -d \
  --name remnawave-manager-bot \
  --restart unless-stopped \
  --env-file .env \
  remnawave-manager-bot
```

### 5. Запуск без Docker (локально)

```bash
# Установка зависимостей
pip install -r requirements.txt

# Загрузка переменных окружения
export $(cat .env | xargs)

# Запуск бота
python bot.py
```

## Использование

1. Найдите вашего бота в Telegram
2. Отправьте команду `/start`
3. Отправьте Telegram ID (число) или username (с @ или без) пользователя
4. Получите полную информацию о пользователе и его подписке

### Примеры запросов

- `123456789` - поиск по Telegram ID
- `@username` - поиск по username
- `username` - поиск по username (без @)

### Поддержка (клиент ↔ менеджер в боте)

1. **Клиент** (пользователь, не из списка менеджеров) пишет боту любое сообщение. Бот ищет его в API Remnawave по Telegram ID и отправляет **каждому менеджеру** в личку сообщение с данными клиента (UUID, статус, трафик, дата истечения) и текстом обращения, а также кнопку **«Ответить»**.
2. **Менеджер** нажимает «Ответить» и пишет следующим сообщением в боте — этот текст пересылается клиенту в личку бота от имени поддержки.

Группа не нужна: всё происходит в личных чатах с ботом.

## Получаемая информация

### О пользователе:
- UUID и Short UUID
- Username и Email
- Telegram ID
- Статус (ACTIVE, DISABLED, LIMITED, EXPIRED)
- Тег и описание
- Статистика трафика (использовано, лимит, всего использовано)
- Даты (создание, обновление, истечение)
- Информация о подключениях
- Активные сквады

### О подписке:
- Статус активности
- Количество дней до истечения
- Статистика трафика
- URL подписки

### О привязанных устройствах:
- HWID каждого устройства
- Платформа и версия ОС
- Модель устройства
- User-Agent
- Даты добавления и обновления

## Безопасность

- Доступ к боту ограничен списком разрешенных ID менеджеров
- Все запросы к API Remnawave требуют авторизации через Bearer token
- Бот работает от непривилегированного пользователя в Docker

## Устранение неполадок

### Бот не отвечает

1. Проверьте, что контейнер запущен: `docker ps`
2. Проверьте логи: `docker logs remnawave-manager-bot`
3. Убедитесь, что `BOT_TOKEN` правильный
4. Проверьте, что ваш ID есть в `ALLOWED_MANAGER_IDS`

### Пользователь не найден

1. Убедитесь, что пользователь существует в системе Remnawave
2. Проверьте правильность введенных данных (ID или username)
3. Проверьте доступность API Remnawave
4. Проверьте правильность `REMNAWAVE_API_TOKEN`

### Ошибки API

1. Проверьте `REMNAWAVE_API_URL` (должен быть без слеша в конце)
2. Проверьте `REMNAWAVE_API_TOKEN` (должен быть валидным)
3. Проверьте доступность API Remnawave
4. Посмотрите логи для деталей ошибки
